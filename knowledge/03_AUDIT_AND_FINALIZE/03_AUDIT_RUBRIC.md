# Script Audit Rubric

Canonical source for audit modes, scoring, verdicts, failure modes, and required audit output.

## Purpose

This rubric exists to validate whether a Short meets the channel's standards before production. Be strict on identity, hook, payoff, retention, and any Stage 1 mental model/framework that was supplied. Calibrate against Anchor Short References as the active voice target. Do not force extra mental-model depth during audit; verify only that any supplied model is natural.

## Lite Audit (15-Minute Weekly Use)

**When to use:** every script, every week, by default. The full 12-category rubric below is for the first 10 Shorts (calibration) and monthly review. For routine weekly auditing, use the Lite version — it is calibrated to catch 90% of issues in 15 minutes.

### The Lite Audit

Run the script against these two passes:

**Pass 1 — 8-Gate Lite Pass (8 yes/no checks, ~5 min):**

1. Hook lands in first 1–5 seconds and creates the strongest available story question? Yes / No — a declarative, stack, sourced scene-first conflict, or viewer-world comparison may pass. A factually correct abstract/receipt-first opening fails when a pictureable carrier would create materially stronger curiosity.
2. Simple curiosity gap? Yes / No
3. Clear payoff? Yes / No
4. Relatable or surprising angle? Yes / No
5. Easy to follow at scroll speed? Yes / No — includes vocabulary, phrase-level clarity, and mental-math/reference stability. More than one numerical relationship in the first five seconds, or adjacent numbers that switch denominator/reference group without an explicit reset, = No.
6. Retention flow holds (new beat every 3–6 sec)? Yes / No
7. Viewer feels smarter? Yes / No
8. Carries our identity (complete Indian English register, our voice, our visual lane)? Yes / No

**Gate:** all 8 must be Yes. Any No → rewrite that specific element before moving to Pass 2.

**Pass 2 — Consolidated Failure-Mode Spot-Check (19 checks, ~15 min):** *(de-duplicated from the former 24-mode list — overlapping modes were folded into their parent so each row is a genuinely independent check the auditor can run deeply, not a rubber stamp. Covers voice/category drift, self-cloning + repetition, hook over-claim, contested claims, unnamed authority, mechanism collapse, context concreteness, final-rule accuracy, word-level vocabulary, phrase-level instant comprehension, and citation integrity.)*

For each, ask one question. If yes to any, rewrite that element.

1. **Forced Mental Model** — did the script add a broader principle not supported by the Stage 1 model/topic? Yes = fail framework integrity and route to repair or Stage 1.
2. **Fake Depth** — does the named principle apply to fewer than 3 other situations? Yes = the principle is fake.
3. **Overexplaining** — could we remove a sentence without losing meaning? Yes = remove it.
4. **Adjacent-Category Drift** (three sub-tells — any one = fail, rewrite tone):
    (a) **Documentary/podcast/news drift** — does this read like a long-form documentary explainer / podcast clip / news anchor?
    (b) **Headline-Style Summarization** — does this read like a Wikipedia compression in our voice? Rewrite native.
    (c) **Hype-Channel Drift** — any banned word, fake outrage, or reaction acting? Strip it.
5. **Forced Sophistication** — did the script add unearned density, essay-like polish, or intellectual weight the topic does not need? Would a sharp friend actually say this aloud? No = simplify.
6. **External-Creator Clone Drift** — could this be re-credited to another known short-form channel without anyone noticing? Yes = change hook, framing, or close.
7. **Polished Explainer Drift + Narrative-Force Regression** — factually clean, but sounds like a documentary/article paragraph rewritten into short lines, or leaves a stronger sourced story asset unused? Run the **4-test detector**:
    (a) **Hook:** quote it verbatim. A single-line declarative, 2-3 line stack, or story-led conflict are valid. It fails if flat, uses receipts where a stronger sourced treatment is available, or line two neither proves, sharpens, nor opens the causal promise.
    (b) **Spoken cadence (restrained):** quote 2 lines whose cadence is clearly spoken — short declaratives, natural-verbal syntax, not written-formal. Restraint is correct. The script must NOT contain reaction-acting tics ("Bas.", "Right?", "But honestly?", "Sounds boring.") or any Hindi/Hinglish. BOTH written-formal drift AND hype-tic drift are failures.
    (c) **Article-feel / paragraph-compression test** ⚠️ **mandatory — check twice; most commonly missed:** paste the 3 densest consecutive lines of the reveal beat as a paragraph. Does it read like a long-form documentary-explainer / magazine-article / Wikipedia-summary paragraph? Also fail if any beat contains a paragraph of 2+ sentences stacked together, unrelated cognitive anchors in one line, or written connectives like "While X, Y" / "In addition to" that overload first listen. Yes = fail; break into vertical spoken thoughts and route to Stage 3 Part B rewrite regardless of other scores. (This test absorbs the former standalone Article-Feel and Paragraph-Compression modes; the joined paragraph + verdict must be quoted in the audit output.)
    (d) **Narrative-force + Pictureability test:** name the narrative proposition, strongest source-backed asset, concrete story carrier, keeper line, and best available story spine. Are they working together, or replaced by something demonstrably more watchable? A research finding, thesis, or mechanism is not a pictureable asset merely because it is true. State what the editor can show without writing the thesis on screen. When the mechanism depends on comparison, selection, or relative price, confirm the carrier shows the options or reference point. Merely mentioning an object or assigning it as decorative B-roll while the spoken script remains abstract = fail.

    Any fail on (a)-(d) = route to Stage 3 Part B rewrite, regardless of how clean the structure and facts are. The auditor must paste the evidence inline; un-evidenced "passes" are auto-fail.

8. **Hook Over-Claim (survivability)** — does the hook claim MORE than the body's facts support (a prototype called "the iPhone", a partial trend called a full reversal, an absolute where the truth is "most of the time")? Yes = soften the hook to what the facts support. A single-line hook is valid and is NOT a defect; the test is accuracy, not line count. For this audience an over-claimed hook makes the correction the top comment.
9. **Underdeveloped Surface Logic** — for Real Reason and Sharp Contradiction lanes: did the script build the viewer's intuitive belief fully before contradicting it? Fewer than 3 specific reasons why the viewer's belief made sense before the contradiction = fail. Yes = route to rewrite; expand the Setup beat to validate the viewer's position before breaking it.
10. **Mechanism Collapse / Causal-Role Blur / Unstaged Behaviour** — if the topic contains two or more mechanisms, are their roles and sequence unclear? If the mechanism concerns a human choice, incentive, trust signal, price, or habit, does the script explain it without showing what the person sees, does, skips, or chooses? Either failure = rewrite using a causal-role map and at least one observable behaviour chain. Explicit labels are optional.
11. **Punchline Dilution / Ending Compression** — does any line follow the final punchline, or does the close contain a line whose removal makes it land harder without losing meaning? Yes = cut it.
12. **Final-Rule Oversimplification / Single-Cause Overclaim** — does the script replace one wrong belief with a new oversimplified belief, OR claim its mechanism is "the whole / the only / the real reason" when other causes also matter? Run the **competing-cause test**: if you removed the named mechanism, would the outcome still be *partly* explained by other real causes (margin, ticket economics, regulation, timing, luck)? If yes, then "the whole business model", "that is the only reason", "this is why X won" is an overclaim. Yes = rewrite the payoff to an honest both/and ("The margin makes it profitable. The smell makes it sell."). The script must state the accurate real rule before the punchline; a memorable line is not a licence to overstate a single cause.
13. **Labelled-Not-Concrete Indian Context** — where Indian context is natural, does the script merely label India / terrace / desi / local without showing a lived condition, familiar contrast, or practical behaviour? Yes = expand or replace the labelled context.
14. **Vocabulary Drift (word-level Comprehension)** — does the script contain any word that would require a dictionary lookup for a Class 10–12 educated Indian metro reader without an immediate plain explanation? Default to one necessary technical term; additional essential terms pass only when each is explained in the same line or the next short line with no intervening idea. Run the Plain Word Test from `../00_SHARED_KB/LANGUAGE_AND_VOICE.md § Reader Benchmark`. *(This is the word-level half of one "Comprehension" check; Mode 18 is its phrase-level half.)*
15. **Self-Clone / Sameness Drift + Thesis-Hammer / Repetition** — two folded tells of the same "is the script repeating a pattern" check:
    (a) **Self-clone / sameness:** strip the proper nouns and compare the skeleton against the last several Shorts in `../00_SHARED_KB/SLATE_LEDGER.md`. Do the hook shape, the pivot phrasing, AND the closer shape all match a recent script? Yes = the slate is cloning itself. Route to rewrite; rebuild from the topic's own contradiction. If the two-part inversion close ("X is not about A, it is about B") was used in the last two Shorts in the ledger, this one must close differently.
    (b) **Thesis-hammer / repetition:** is the script's one idea restated without adding a new causal role, image, or consequence? Yes = cut it. Genuine progression along the story spine is not repetition.
16. **Contested-Claim-as-Settled** — does the script state a genuinely contested academic or historical interpretation as plain fact (e.g., Sen's famine-entitlement reading, colonial-policy causation)? Yes = attribute it as a named argument ("X argued, in a reading still debated, that…").
17. **Unnamed / Vague Authority** — does any claim lean on authority that is not a named, checkable source? This covers two forms: (a) the classic "studies show", "researchers found", "experts say"; AND (b) **vague rhetorical authority** used as an evidentiary setup — "every analyst", "most economists agree", "the usual explanation is", "everyone knows", "people say". Form (b) is NOT exempt as "just a rhetorical pivot" — if it is doing the work of establishing what the smart/standard view is, it must be either named ("the standard margin explanation") or rephrased so it claims no authority it cannot back. Yes = name the source, attribute the view concretely, or cut the appeal.
18. **Source-Register / Instant-Comprehension / Mental-Math Drift (phrase-level Comprehension)** ⚠️ **route-to-rewrite, not a note** — does any line use report-style phrasing, unnecessary spoken citation furniture, or mental math an average Indian viewer would not process instantly? Specific triggers: unfamiliar author/journal names that add no spoken value; a common word used as a proper noun; an unexplained institution; archive/report register; more than one numerical relationship in the first five seconds; adjacent ratios/percentages that switch denominator/reference group without an explicit reset; finance terms used imprecisely. Any failure = route to rewrite: move full citation on-screen, speak the study shape where appropriate, simplify/move numbers, and preserve the exact fact. *(This is the phrase-level half of one "Comprehension" check; Mode 14 is its word-level half.)*
19. **Citation Misdescription / Integrity** ⚠️ **route-to-rewrite — re-verify, do not trust the upstream note** — for every load-bearing claim that names a source, the auditor must independently re-fetch the source and confirm all four: (1) the **outlet/journal name** is exactly right ("i-Perception" ≠ "Perception"); (2) the **author and year** are right; (3) the **document type** is right — a review or commentary is NOT a primary study, so "a study found" must point to the actual experiment (and name its sample/location if the claim rests on it); (4) the source supports the **exact claim** made, not an adjacent one. A high-tier source described wrongly still fails — "tier 1, link present" is NOT verification of the citation, only of its existence. Per `../00_SHARED_KB/SOURCE_AND_FACT_RULES.md § Citation Integrity`. **Re-fetch discipline:** a full re-fetch is mandatory when the claim is time-relative, contested, Sensitive-tier, or named in the audit; when a Stage-1 dossier row already carries a resolving URL + verbatim quote + accessed date and the claim is none of those, a freshness/contested check suffices instead of a full re-fetch (per `../00_SHARED_KB/SOURCE_AND_FACT_RULES.md § Citation Integrity`). Any miss = route to Stage 3 Part B: reframe the citation to what can be confirmed, or cut the claim. The auditor records, per cited source, the re-verified outlet/author/year/type and the one specific finding it supports. This record IS the Source Dossier (`../00_SHARED_KB/SOURCE_INTEGRITY_AND_CLARITY_GATE.md § The Source Dossier`): outlet | author | year | type | verbatim finding (quoted) | resolving URL | accessed. A row that cannot be filled because the source will not fetch = the claim is unverified; soften or cut it (no-fetch, no-lock). Sensitive-tier scripts require this dossier to be filled by an independent verification pass, not the auditor.

**Source Integrity and Audience Clarity Gate (mandatory, binary, blocks production):** before any lock, the script must pass all five pillars of `../00_SHARED_KB/SOURCE_INTEGRITY_AND_CLARITY_GATE.md` — source integrity, mechanism honesty, authority placement, instant audience clarity including mental math, and temporal freshness. Any pillar FAIL = the script cannot lock.

**Hard-number verification gate (mandatory):** every specific figure presented as fact (rent totals, outlet counts, applicant numbers, dates, "X% of …") must carry a source + date in the fact-check notes before the script can lock. Unverifiable numbers are softened or cut, not published.

**Story-force + concrete-carrier gate (mandatory):** factual specificity is not sufficient hook strength. Identify the strongest source-backed narrative asset and the strongest concrete story carrier available. The carrier may be a sourced scene or a clearly illustrative viewer-world scene/comparison that invents no specific event, motive, quote, or measured result. If the script opens with abstract explanation, receipts, ratios, caveats, or source exposition while leaving a materially stronger pictureable carrier unused, Gate 1 and Mode 7 (Polished Explainer Drift) fail.

**Story-Shape Lock (mandatory, binary, blocks production):** in addition to the gate above, run all four checks of `../00_SHARED_KB/STORY_SHAPE_LOCK.md` at lock — carrier used in the first third, open earns a Google, payoff sequenced/single-true-cause, thesis once + close not hammered. **Any FAIL caps the verdict at Generic and forces Rewrite Trigger = Yes**, with the same blocking power as a Source-Integrity pillar. This is the structural lever that prevents a safe-but-flat script from grading Strong.

**Temporal-freshness gate (mandatory):** every use of "latest", "currently", "today", "this year", or equivalent is re-fetched at final lock. Exact older-period facts used as present proof must also be checked against the newest comparable primary evidence. A superseded "latest" or stale present-proof claim is a production blocker.

**Current-function freshness + scope gate (mandatory):** exact older-period evidence used as present-day proof is checked against the newest comparable source. Evidence cannot silently expand from one company to an industry, one experiment to everyone, or observed effect to deliberate intent.

**Best-available-version gate (mandatory):** when a materially different safe treatment exists, compare at least two viable full treatments. A merely compliant treatment fails if another is clearly stronger on hook pull, spine clarity, causal progression, cognitive load, payoff, and ending force.

**Packaging readiness (check at lock):** the locked package carries a title option, a thumbnail stat/phrase (the single shocking number or image leads — not buried in line three), and a first-frame note. Strong scripts now win or lose on packaging; do not pass a script to the editor brief without it.

**Verdict format (one line):**
*"Lite Audit — 8/8 pass + 0/19 failures detected (Article-feel + self-clone evidence quoted) -> route to Stage 3 Part B Direct finalization."* OR *"Lite Audit — fail on point [N] + failure mode [X] -> route to Stage 3 Part B with this fix list."*

The "Article-feel + self-clone evidence quoted" caveat is mandatory: a pass cannot be recorded unless the auditor has pasted the verbatim quotes required by failure mode 7 (a)-(c) and the self-clone/repetition checks into the audit output.

### Viral Social Commentary Overlay (Mandatory When Applicable)

Run this overlay when the script declares Viral Social Commentary OR the topic shape matches `../00_SHARED_KB/VIRAL_SOCIAL_COMMENTARY.md § When This Lane Applies`, even if the script incorrectly declared another lane.

The script must quote or clearly show all six:

1. Viral object + price / pay / time / rule in the hook.
2. What the service / behaviour actually does.
3. Side A steelman with a concrete benefit or stakeholder.
4. Side B steelman with wage / price / behaviour math.
5. Direct viral evidence when the original wording is the proof.
6. Discomfort + specific viewer-question close.

Any missing item = **Adjacent-Category Drift** (Failure Mode 4) and Rewrite Trigger = Yes. Do not pass a Viral Social Commentary topic as Hidden India, Smart Money, or Sharp Contradiction just because the script has a clean observation. A clean observation without the debate payload is the failure.

Specific auto-fails:
- "Worker pay ka question still matters" without the job-listing quote or wage math.
- "India did not invent the helper; it priced the helper" as the final payoff.
- "Market ka jugaad ab rate card pe aa gaya" as a Hinglish close.
- Any ending without a stakes-naming viewer question.
- Any script that turns the viral object into a timeless insight and removes the live internet debate.

**Op-Ed Aphorism Test (mandatory for this lane):** read the script's closing 2 lines aloud. Do they sound like a New York Times sub-head or a magazine kicker ("India did not invent the helper; it priced the helper" / "Market ka jugaad ab rate card pe aa gaya")? Yes = auto-fail. The closer in this lane is the viewer question, not the host's aphorism. See `../02_SCRIPT_CREATION/04_SCRIPT_DRAFTING_GUIDE.md § Beat 6 The Payoff + Transferable Insight § Op-Ed Aphorism test (Shape B)`.

### When to Use the Full Rubric Below Instead

- First 10 Shorts of the channel (calibration period)
- Monthly review of the last 4 weeks of Shorts
- Any script flagged by the Lite Audit twice and still failing
- Any sponsored Short or brand integration
- Any current-affairs Short with named individuals or specific claims

The full 12-category rubric is below. It is more rigorous but takes 40–60 minutes. Do not use it for routine weekly work — it will be skipped.

*Note: the 12 categories in the full rubric are distinct from the 19 failure modes in the Lite Audit Pass 2. Both apply; they measure different things.*

## Scoring System

Each Short is scored across **12 categories on a scale of 1–10**.

| # | Category | What it measures |
|---|----------|------------------|
| 1 | Channel Identity Fit | Could only have been made by this channel — complete Indian English, sharp, internal-voice-original. |
| 2 | Hook Strength | Hook stack lands in 1–5 seconds, specific, scroll-stopping. |
| 3 | Curiosity Gap | Viewer instantly wants the answer. |
| 4 | Payoff Strength | The Short delivers a clean reveal / takeaway / observation. |
| 5 | Retention Flow | New beat every 3–6 seconds, no flat middle. |
| 6 | Easy-to-Understand | Clear at scroll speed. No jargon, no prerequisites. |
| 7 | Originality of Angle | Our framing is sharper than what other channels would do. |
| 8 | Indian Relevance (conditional) | Natural Indian angle if applicable; otherwise N/A. |
| 9 | Language Quality | Complete Indian English, clean, native. Sounds spoken. No Hindi or Hinglish. No written-language rhythms. |
| 10 | Tone | Calibrated for topic. Host could deliver naturally. |
| 11 | Ending | Memorable payoff line, identity-bearing close. |
| 12 | Fact Risk | All claims verifiable and defensible. |

## Category 1: Channel Identity Fit

**10/10:** Could only have been made by this channel. Complete Indian English. Sharp. Voice originality is internal, not borrowed.

**5/10:** Could plausibly belong to this channel but could also fit 2–3 others.

**Failure:** Sounds like a generic Hinglish fact channel OR like a long-form documentary-explainer Short.

## Category 2: Hook Strength

**10/10:** First 1–5 seconds. One sharp line, stack, or source-backed scene-first conflict. Creates a concrete story question with low cognitive load. The viewer cannot scroll past it.

**5/10:** Hook is acceptable but feels familiar / not memorable.

**Failure:** Opens with greeting/generic context; fails to create curiosity; or leads with multiple ratios/receipts when a materially stronger source-backed story opening is available.

## Category 3: Curiosity Gap

**10/10:** The viewer immediately wants the answer — "wait, why?" or "what actually happened?" feels mandatory.

**5/10:** Curiosity is mild. The viewer might watch on autopilot.

**Failure:** No curiosity at all. The Short is a flat statement of fact with no pull.

## Category 4: Payoff Strength

**10/10:** The Short delivers a clean reveal, useful insight, sharp observation, or named principle. The viewer leaves with something specific to repeat.

**5/10:** Payoff exists but lands soft. Slightly anticlimactic.

**Failure:** No real payoff. The Short stops without resolving the curiosity gap, or summarises instead of paying off.

Note: Payoff is still a core gate. When Stage 1 supplied a mental model/framework, a great payoff should make it feel concrete and repeatable. When Stage 1 supplied none, do not treat the absence as a defect.

**Payoff-accuracy check (mandatory):** Does the payoff create a new oversimplification while correcting the old one? If the real answer is more nuanced than a single line (e.g., the actual rule applies to a mechanism, not a colour or category), the payoff must include one additional line stating that nuance before the punchline. A memorable but inaccurate payoff scores ≤6 regardless of how clean the line sounds.

## Category 5: Retention Flow

**10/10:** New information / new beat every 3–6 seconds. No flat sections. Each sentence advances or rewards.

**5/10:** Retention drops in one section, usually the middle.

**Failure:** Multiple flat sections. The viewer would scroll away.

## Category 6: Easy-to-Understand

**Benchmark:** The assumed viewer is a **Class 10–12 educated Indian metro reader** — not an engineer, not an MBA. Any word that would require a dictionary lookup for this reader is a defect, whether it is a technical term or just a high-register everyday English word ("incumbent", "oxidise", "rancid", "proliferate", "inert"). The fix is either a plain replacement or an immediate inline definition. See `../00_SHARED_KB/LANGUAGE_AND_VOICE.md § Reader Benchmark`.

**10/10:** Clear at scroll speed. Plain or immediately defined language, no unnecessary spoken citation furniture, no unexplained institution, and no mental-math/reference-switch friction. The first five seconds carry at most one numerical relationship.

**5/10:** Mostly clear, but contains 1–2 lines that need a second listen, OR 1–2 high-register words that are undefined.

**Failure:** Reads like a documentary or academic explainer — too dense for the platform — OR contains multiple undefined high-register words that an average Indian viewer would not know.

## Category 7: Originality of Angle

**10/10:** Our framing is sharper than what other channels would do. The hook, sequence, or close has identity.

**5/10:** Topic is fine; angle has been seen before.

**Failure:** Reads like a translation of a common Western explainer or like an existing Indian creator's exact framing.

## Category 8: Indian / South Asian Relevance (Conditional)

**Important:** If the topic doesn't naturally invite Indian relevance, score 10/10 with note "N/A — topic doesn't require Indian context."

**10/10:** Indian relevance is meaningful, concrete, and naturally integrated, OR genuinely not applicable. Concrete means a lived condition, familiar contrast, or practical behaviour the viewer can picture.

**5/10:** Indian relevance is present but feels added rather than essential, OR is missing where it should have been natural.

**Failure:** Forced, tacked on, politically risky, or labelled without concrete lived detail.

## Category 9: Language Quality

**10/10:** Complete Indian English, native, sharp. Every line sounds spoken, not written.

**5/10:** Mostly clean but contains 1–2 lines with written-language rhythms or register drift.

**Failure:** Hindi or Hinglish in the script OR written-language rhythms throughout OR register drift into documentary/academic.

## Category 10: Tone

**10/10:** Calibrated correctly for the topic. Energy level matches content. Host could deliver this naturally and casually.

**5/10:** Tone is mostly correct but 1–2 lines drift toward teacher / journalist / hype-channel modes.

**Failure:** Reads like a different category — Hinglish entertainment, documentary explainer, motivational reel, news anchor.

## Category 11: Ending

**10/10:** Lands with a sharp, memorable line in our identity voice. The viewer takes something specific.

**5/10:** Ending is acceptable but not memorable.

**Failure:** Generic, includes "like and subscribe," motivational drift, or simply stops without landing.

## Category 12: Fact Risk

**10/10:** All claims are verifiable and defensible. No oversimplifications. No unsupported statements.

**5/10:** Most claims solid but 1–2 require careful verification before publishing.

**Failure:** Multiple claims unsupported, oversimplified, or factually risky.

## Mental Model / Framework Integrity

Every submitted Short may carry the natural mental model/framework approved in Stage 1. If one is present, audit whether it is natural and not forced.

- **Pass:** Clear Stage 1 mental model appears in the package and the script does not force it, OR no model was supplied and the script does not invent one.
- **Fail:** Fake model, or model bolted onto the ending without support.

Do not invent a new model during audit. If the Stage 1 brief did not supply a model, write `N/A — optional bonus`.

## Failure Mode Definitions

The canonical definitions are the 19 questions in Lite Audit Pass 2 above. Do not maintain a second local explanation list here; duplicate definitions drift. When producing an audit, quote exact script evidence for every triggered mode and route the fix to Stage 3 Part B.

## Calibration Anchors

### Hook Score Calibration

**9–10/10 (hook stack — one valid shape):**
"The hottest thing humans ever made... was not a nuclear bomb. / It was not even the Sun. / It happened inside a 27-kilometre tunnel under Europe."
Why: Three lines, kills two obvious answers in lines 1-2, reveals location only in line 3. Mechanism deliberately withheld. Pure curiosity gap.

**9–10/10 (hook stack — Real Reason / Hidden India everyday topic):**
"Transparent water tanks look smarter, right? / You can see the water level. You can see the dirt. / But on an Indian terrace, that logic fails."
Why: Sets up the viewer's intuition, validates it, then contradicts it with a place anchor. Short declaratives with natural spoken cadence.

**9–10/10 (single-line — fully valid for any lane; often the more premium choice):**
"Britain's most decorated female spy in the Second World War was an Indian princess."
Why: Declarative, specific, complete Indian English, instantly overturns assumption.



**7–8/10:** "Your protein brand might be lying about its numbers. Most are never independently tested."
Why: Sharp, relatable, sets up curiosity with a specific — no host-credential tic (compare the banned "even I was shocked" framing in § Banned Creator Tics).

**5–6/10:** "Global supply chains have a priority list."
Why: Conceptual; needs the next sentence to fully grip.

**Failure:** "Did you know Nokia used to be huge?"
Why: Question form, banned opening, no specificity.

### Payoff Score Calibration

**9–10/10:** "Nokia didn't lose to the iPhone. Nokia lost to Nokia."
Why: Sharp, memorable, lands the reveal as the line viewers repeat.

**9–10/10:** "This was never a Diet Coke shortage. India is just lower on the list."
Why: Conclusive, identity-bearing, complete Indian English.

**9–10/10 (Viral Social Commentary — viewer-verdict close):** "Tell me, would you use this service for yourself or for your parents? Share this with somebody who needs to see this."
Why: Names the specific stakeholder ("yourself or your parents"), is answerable as yes/no/depends, and the share prompt names a real audience. Not a writerly aphorism — the audience verdict is the close. Single-line aphorisms like *"India did not invent the helper; it priced the helper"* score ≤6 in this lane (auto-fail per the Viral Social Commentary overlay).

**7–8/10:** "Mosquitoes are reading a signal your skin sends out. You just aren't part of the conversation."
Why: Smart observation, slightly poetic, lands clean.

**5–6/10:** "So that's how the supply chain works."
Why: Summary, not payoff.

**Failure:** "What do you think? Let me know in the comments."
Why: Generic CTA, no specific stakes.

### Mental Model / Framework Calibration

**Strong model:** "History remembers people who fit narratives."
**Strong model:** "Constraint produces precision."
**Weak / likely reject:** "Bamboo got reclassified from tree to grass" if no broader transferable pattern is found.

## Verdict Labels

After scoring, assign one verdict:

- **Strong (8.0+ average):** Ready for Stage 3 Part B Direct finalization. Minor verification only.
- **Good but needs polish (7.0–7.9):** Stage 3 Part B applies specific line edits, verifies facts, and locks the final script.
- **Generic (5.5–6.9 OR Polished Explainer Drift detected OR Viral Social Commentary overlay failed):** Major rewrite needed.
- **Off-brand (any score, fails identity):** Reject regardless of polish.
- **Fact-risky (any score, fails fact-check):** Stage 3 Part B verifies, softens, or cuts the risky claim before the script can lock.
- **Reject (under 5.5):** Route to Stage 3 Part B for AI reframe/regeneration if defensible; drop when no clean AI reframe exists. (Any verdict — including Strong — is overridden to **DROP** by the Final Disposition gate below if the idea fails the Idea-Strength re-score. Clean writing does not earn a PASS.)

If a script is factually correct, cleanly structured, and easy to understand but triggers **Polished Explainer Drift**, it is not Strong. Route to Stage 3 Part B Rewrite mode. The fix is a voice/retention rewrite, not a fact rewrite.

If a script passes every safety gate but **fails any Story-Shape Lock check** (`../00_SHARED_KB/STORY_SHAPE_LOCK.md`), it is **not Strong** — the verdict is capped at **Generic** and Rewrite Trigger = **Yes**, no matter how high the execution scores. A safe-but-flat script is exactly what this cap exists to stop.

If a live viral debate is factually clean but missing the two-sided debate payload, wage / price math, direct viral evidence, or viewer-verdict close, it is not Strong. Route to Stage 3 Part B Rewrite mode. The fix is a lane rewrite, not a line polish.

## Final Disposition — PASS / FIX / DROP (mandatory, runs last)

The verdict labels above grade **execution**. They do not, by themselves, decide whether the *idea* is worth producing — and a clean rewrite can carry a weak idea to a passing verdict. After scoring (and after any rewrite), assign one disposition:

- **PASS** — production-ready: strong idea AND clean execution.
- **FIX** — real potential, execution flawed → rewrite (the existing Rewrite-Trigger path).
- **DROP** — not worth producing → stop. Give a one-line reason. Do NOT rewrite to rescue it.

### Idea-Strength re-score (the teeth of the gate)

Clean writing must never carry a weak idea to PASS. Re-run these five concept checks on the FINAL script's core proposition. Keep identity-bearing proper nouns when the person, place, company, or object is the actual novelty. Strip proper nouns only for the separate self-clone/skeleton check; stripping them here would incorrectly erase the idea in identity-led history and business stories.

1. **Curiosity gap** — would a viewer actually Google this?
2. **Repeatable payoff** — one line they would send a friend?
3. **Surprise / novelty** — or is it a saturated explainer the audience has already seen?
4. **Mechanism strength** — a real hidden reason, not just a fact?
5. **Channel-identity fit** — sharp, insight-led; not a generic explainer?

**Hard rule:** if the stripped-down idea fails **2 or more** of these, the disposition is **DROP** — even if every execution gate scores ≥8. A cleanly written but weak idea is a DROP, not a FIX. This override sits above every verdict label and every rewrite route. Rewrite is only for execution defects on a strong idea.

### DROP reason one-liners (pick the closest, edit to fit)

- "DROP — Cleanly written, but a generic explainer; does not match the channel's sharp, insight-led format."
- "DROP — Factually valid but weak audience pull and low novelty."
- "DROP — Does not fit the channel's identity or expected viewer interest."
- "DROP — Even after rewrite, the mechanism is weak and the payoff does not land."
- "DROP — Useful information, but not a strong short-video idea."

A DROP is logged (one line) in the final package so the pattern is visible over time. The pipeline does not loop a DROP back into endless rewrites; it returns to 01_TOPIC_EVALUATION only if a genuinely different angle exists.
