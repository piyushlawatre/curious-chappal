# Story-Shape Lock

This is a **binary, blocking gate** — a sibling to the Source Integrity and Audience Clarity Gate. It runs at Stage 3 lock, after the audit modes and alongside the five source/clarity pillars. Every check below is PASS or FAIL. **Any single FAIL caps the audit verdict at "Generic" and forces Rewrite Trigger = Yes** — exactly the blocking power a Source-Integrity pillar FAIL has.

## Why this gate exists

The pipeline used to enforce **safety** with hard locks (overclaim, vague authority, denominator switch, stale fact, citation error all *block* a lock) but enforce **story** only with advice (story-before-receipts, narrative-force, best-available-version all lived inside the 24-mode wall and none capped the verdict). The result was a structural bias: a safe-but-flat script sailed through every blocking gate and graded Strong, while a vivid-but-unsafe script got hard-blocked — so under any ambiguity the system converged on safe-and-flat. This gate removes that asymmetry. Story now blocks with the same force as safety.

## The four checks (all binary)

- [ ] **1. Carrier used.** The Stage-1 concrete story carrier (or a demonstrably stronger one found in drafting) appears in the **first third** of the script and actually carries the mechanism — not as decorative B-roll over an abstract spoken track. A sourced thesis narrated over a relevant image is a FAIL. **PASS / FAIL.**
- [ ] **2. Open earns a Google.** The first 1–2 lines create a **story question a viewer would actually search** ("wait, why / what happened?"), not a stat or receipt a viewer would simply nod at. An abstract, receipt-first, ratio-first, or caveat-first opening when a materially stronger pictureable carrier was available is a FAIL. **PASS / FAIL.**
- [ ] **3. Payoff is sequenced or single-true-cause.** The payoff lands as a **sequenced reveal or a single genuinely-true cause** — never a flat additive list ("and also, and also"), and never a single-cause overclaim where other real causes also matter (run the competing-cause test). **PASS / FAIL.**
- [ ] **4. Thesis stated once + once at close, not hammered.** The one idea is stated once where it lands and once at the close — not repeated through the body without adding a new causal role, image, or consequence. **PASS / FAIL.**

## Gate result

```
STORY-SHAPE LOCK
Check 1 — Carrier used (first third, carries mechanism):     PASS / FAIL
Check 2 — Open earns a Google (story question, not a stat):  PASS / FAIL
Check 3 — Payoff sequenced / single-true-cause (not flat):   PASS / FAIL
Check 4 — Thesis once + close, not hammered:                 PASS / FAIL

GATE: PASS only if all four are PASS.
Any FAIL = audit verdict capped at "Generic" + Rewrite Trigger = Yes. The script routes to Stage 3 Part B rewrite exactly like a safety-pillar failure.
```

This gate cannot be waived for an otherwise clean script. A factually correct, safe, easy-to-follow script that opens on a receipt, buries its carrier, lists a flat payoff, or hammers its thesis is exactly the flat script this gate exists to stop from grading Strong.
